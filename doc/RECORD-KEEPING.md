# Record keeping — what is recorded, where, and its lifecycle

- **Scope:** the record types this workspace maintains, their locations, immutability
  rules, and lifecycles. Authorities: ADR-0014 (handoff), ADR-0017 (coordination),
  ADR-0018 (message lifecycle), `Enconet/coordination/TEAM_PROTOCOL.md`,
  `doc/GIT_CONVENTIONS.md`.
- **Owner:** shared (either agent under the coordination protocol); lifecycle rules
  change only via ADR.
- **Update trigger:** a new record type, or an ADR changing a lifecycle rule.

## Record types

| Record | Location | Mutability |
|---|---|---|
| ADRs | `Enconet/decisions/` (register: `decisions/README.md`) | Immutable once accepted; superseded, never edited |
| Handoffs | `Enconet/handoffs/YYYY-MM-DDTHHMMSSZ-<short-sha|nogit>.md` | Immutable; `HANDOFF.md` is the mutable pointer, published only after the record validates |
| Coordination messages | `Enconet/coordination/messages/` (active) → `coordination/archive/` | Immutable, author-prefixed (`CC_`/`CX_`); archive only resolved **and** confirmed, via a resolution manifest; silence ≠ confirmation |
| Task claims | `Enconet/coordination/claims/<task>.yml` | Created on claim, released on completion; 24h expiry |
| BOARD | `Enconet/coordination/BOARD.md` | Generated by `scripts/agent_coord.py status --write`; never hand-edited, never authoritative history |
| DATA manifest | `Enconet/sieving/DATA_MANIFEST.json` | Tracked and machine-verified; `sieving/DATA/` itself never tracked (ADR-0002), never modified in place |
| Provenance / divergence log | `Enconet/sieving/PROVENANCE.md` | Append-only table of local modifications |
| Guidance drift manifest | `doc/GUIDANCE_PAIRS.json` | Agent-neutral shared authority; changes proposed via coordination messages |
| Commits | git history, `main` | Tagged per `doc/GIT_CONVENTIONS.md`; agent authorship via `Co-Authored-By` trailers |

## Rules that cut across record types

- **Author prefix = write permission.** Each agent creates/moves only its own
  `CC_`/`CX_` records; neutral records (`TEAM_PROTOCOL.md`, `BOARD.md`,
  `GUIDANCE_PAIRS.json`) are modified under the shared protocol.
- **Blockers need a disposition** (`resolved` / `owner-accepted` / `deferred-until`) —
  never archival-by-acknowledgement (ADR-0018).
- **No record implies success it cannot prove.** Validation states are
  `passed|failed|not-run|unknown`; missing infrastructure is recorded as
  `not-configured`, not invented.
- **Session bracket:** session start reads `HANDOFF.md` → `BOARD.md` → unread
  messages → active claims, then verifies against reality; session end runs the
  user-global `/handoff` skill (mandatory) producing a new immutable record.
